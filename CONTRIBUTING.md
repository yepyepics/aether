# Contributing to Aether

Thank you for considering contributing to Aether!

## Development Flow

1. Fork the repository.
2. Create a branch using one of the following conventions:
   - `feature/<name>`
   - `fix/<name>`
3. Make your changes and keep commits focused.
4. Commit your work with clear, descriptive messages.
5. Open a Pull Request against the main repository.

## Code Requirements

- Ensure the code is formatted and validated before opening a Pull Request.
- Run `npm run lint` before submitting changes.
- Include screenshots in the Pull Request for any UI-related change.

## Pull Request Notes

- Keep Pull Requests scoped to a single purpose whenever possible.
- Reference related issues in the Pull Request description.
- Be explicit about any platform-specific behavior or testing limitations.
